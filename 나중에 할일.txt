-  데이터 마이그레이션(DB 백업)

   1. 데이터 덤프(Backup) 생성:
      현재 사용하는 DB(Render의 PostgreSQL)에서 데이터를 SQL 파일 형태로
  추출합니다. pg_dump라는 도구를 사용하면 아주 쉽게 할 수 있습니다.
   2. 새 DB로 복구(Restore):
      Supabase나 Neon에서 새로운 DB를 만든 뒤, 추출한 SQL 파일을 실행하여
  데이터를 그대로 밀어 넣습니다.
   3. 환경 변수 교체:
      배포 환경(Render의 API) 설정에서 DB 연결 문자열(DATABASE_URL)을 새 DB
  주소로 바꿉니다.

-DB 이동
 * Neon (Serverless PostgreSQL):
       * 장점: Postgres 기반의 서버리스 DB로, 무료 티어에서 영구적으로 사용
         가능합니다.
       * 방법: Prisma와 매우 잘 호환되며 설정이 간단합니다.


-Cold Start 방지
1. 무료 업타임 모니터링 서비스 활용 (가장 추천)
  서버에 5~10분 간격으로 가짜 요청(Ping)을 보내주는 무료 서비스를 등록하면
  됩니다.

   * UptimeRobot (https://uptimerobot.com/):
       * 방법: 무료 계정을 만들고, 서버의 API 주소(예:
         https://your-app.onrender.com/health 등)를 등록합니다.
       * 설정: 5분 간격으로 모니터링하도록 설정하면, 주기적으로 요청이 발생하여
         서버가 절대 잠들지 않습니다.
       * 장점: 설정이 매우 간단하고 안정적입니다.
   * Cron-job.org (https://cron-job.org/):
       * 방법: 무료로 원하는 시간에 특정 URL을 호출하게 할 수 있습니다. 10분마다
         서버를 '핑'하도록 설정하면 됩니다.
