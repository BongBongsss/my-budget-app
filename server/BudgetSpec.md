# 자동화 가계부: 제품 요구 사항 문서 (PRD)

## 1. 프로젝트 개요
**자동화 가계부** 애플리케이션은 지출 내역을 기록하는 수고를 최소화하고, 자동화된 데이터 처리와 시각적 대시보드를 통해 깊이 있는 금융 통찰력을 제공하는 것을 목표로 합니다.

## 2. 핵심 기능

### 2.1. 데이터 입력 및 자동화 (기능 그룹 1)
수동 입력의 실수를 줄이고 금융 데이터를 자동으로 수집하여 시간을 절약하는 것이 목표입니다.

*   **CSV/엑셀 임포트:** 사용자가 은행 거래 내역이나 카드 결제 내역을 업로드할 수 있습니다. 시스템은 이 파일들을 파싱하여 날짜, 금액, 결제처, 거래 유형을 자동으로 추출합니다.
*   **자동 카테고리 분류:** 
    *   키워드 매칭 엔진을 사용하여 카테고리를 할당합니다. (예: '스타벅스' -> '식비 및 음료')
    *   사용자는 특정 결제처에 대해 사용자 지정 규칙을 정의할 수 있습니다.
*   **고정 지출 관리:**
    *   반복되는 지출(구독료, 월세, 보험료 등)을 식별하고 표시합니다.
    *   해당 결제일에 해당 내역이 자동으로 기록되도록 예약합니다.
*   **수동 조정:** 자동 분류가 잘못된 경우 사용자가 내역을 수정하거나 항목을 분할할 수 있어야 합니다.

### 2.2. 분석 및 시각화 (대시보드) (기능 그룹 3)
원시 데이터를 시각적 인터페이스를 통해 실행 가능한 금융 인사이트로 변환합니다.

*   **지출 대시보드:** 
    *   **개요:** 이번 달 총 수입, 총 지출, 순 저축액을 표시합니다.
    *   **지출 내역 분류:** 카테고리별 지출 비중을 보여주는 파이 차트 또는 도넛 차트.
    *   **지출 추세:** 한 달 동안의 일별/주별 지출 패턴을 보여주는 선 그래프 또는 막대 그래프.
*   **비교 분석:**
    *   현재 달의 지출을 지난달과 비교하여 증감률(%)을 표시합니다.
    *   실제 지출과 설정된 카테고리별 예산을 비교합니다.
*   **주요 결제처 통계:** 가장 많은 돈을 지출한 상위 5개 결제처 리스트 또는 차트.
*   **필터링 및 상세 조회:** 날짜 범위(최근 30일, 올해, 사용자 지정) 또는 특정 카테고리별로 대시보드를 필터링할 수 있습니다.

## 3. 기술적 요구 사항
*   **데이터 저장:** 거래 내역 및 사용자 설정은 로컬 또는 클라우드 데이터베이스에 저장되어야 합니다.
*   **파일 처리:** 일반적인 은행 엑셀/CSV 형식(.csv, .xlsx)을 지원해야 합니다.
*   **데이터 내보내기:** 분석된 데이터를 외부 보고용으로 다시 CSV 파일로 내보낼 수 있어야 합니다.

## 4. 사용자 경험 (UX) 목표
*   **깔끔한 인터페이스:** 차트와 리스트의 가독성을 최우선으로 합니다.
*   **원활한 임포트:** 업로드 프로세스는 직관적이어야 하며, 최종 저장 전 미리보기 단계를 제공합니다.
*   **모바일 최적화 (선택 사항):** 이동 중에도 대시보드를 확인할 수 있도록 반응형 디자인을 적용합니다.
